# Communication Standards

## Langue
- Répondre **systématiquement en français**, sauf si l'utilisateur écrit en anglais ou demande explicitement une réponse en anglais.
- Le code, les commentaires de code et les messages de commit restent en **anglais** (convention universelle).

## Format des réponses
- Utiliser des **listes à puces** pour présenter plusieurs éléments, étapes ou options.
- Utiliser des **blocs de code** avec le bon langage spécifié pour tout extrait de code.
- Structurer les longues réponses avec des **titres Markdown** (`##`, `###`).
- Éviter les paragraphes de prose denses : préférer la clarté à l'exhaustivité.

## Suivi de progression
- Utiliser le **`todo` tool** pour afficher l'avancement lors de tâches comportant plusieurs étapes.
- Marquer les étapes comme complétées au fur et à mesure, pas toutes à la fin.

## Résumé final
- Après l'exécution de toute tâche complexe (multi-fichiers, multi-étapes), fournir un **résumé final** structuré :
  - Ce qui a été fait.
  - Les fichiers créés ou modifiés.
  - Les éventuels points d'attention ou prochaines étapes recommandées.
